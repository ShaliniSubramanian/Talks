Designing for People with Disabilities

  1. Visual
  2. Hearing

How people with disabilities use web and mobile experiences

  1. Blind - Screen reader, Screen magnifier, HCM, Custom styles
  2. Hearing - Captions, Description
  3. Motor - Adaptive Keyboard, special mouse, switches, Head Mouse

Designing for Old users

Old users -Challenges

  • Vision Decline - Decreasing ability to focus on near tasks,Contrast sensitivity,Pupil shrinkage
  • Hearing loss
  • Motor- Trembling in the hands , may operate with single fingers
  • Cognitive - shows decline in memory

Device Dependent Events (Developer)

Wrong


<a href="newuser.html" onMouseOver="showMenu()" onMouseOut="closeMenu()">

Correct


<a href="newuser.html" onMouseOver="showMenu()" onFocus="showMenu()" onMouseOut="closeMenu()" onBlur="closeMenu()">
  • Use onFocus, onBlur, onSelect, onChange, and onClick (when onClick is used with link or form elements)
  • onMouseOver, onMouseOut, and onDblClick rely upon the use of a mouse
  • onKeyDown, onKeyUp, etc. rely upon Keyboard

Error Recovery (Developer)

aria-required, aria-describedBy, aria-invalid, role=alert 


<label for="password" class="">Create your password</label>
<input id="password"  aria-invalid="true" aria-describedBy="password-help-error" aria-required="true" type="password" >

Summary:::Top 10 Barriers we frequently see at PayPal

  1. Img element missing an alt attribute (Content Publisher/Developer)

  5. Form does not contain a submit button (Designer/Developer)
  6. Lang Attribute Missing (Developer)
  7. Lack of Labels (Designer / Developer)
  8. Events bound to items that are not discoverable via keyboard (Developer)
  9. Error Recovery (Developer)
  10. Fake Elements (Developer)
